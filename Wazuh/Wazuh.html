¿Qué es Wazuh?

Wazuh es una plataforma de seguridad de código abierto que proporciona detección de intrusiones, monitoreo de integridad de archivos, análisis de logs y cumplimiento en tiempo real.

Basado en el stack ELK (Elasticsearch, Logstash, Kibana) y con agentes ligeros, Wazuh protege infraestructuras físicas, virtuales y en la nube mediante un enfoque unificado de seguridad.

# Instalación del agente Wazuh en Linux
curl -so wazuh-agent.deb https://packages.wazuh.com/4.x/apt/pool/main/w/wazuh-agent/wazuh-agent_4.5.0-1_amd64.deb
sudo WAZUH_MANAGER='10.0.0.2' dpkg -i ./wazuh-agent.deb
sudo systemctl daemon-reload
sudo systemctl enable wazuh-agent
sudo systemctl start wazuh-agent
# Verificar estado del agente
sudo systemctl status wazuh-agent

Evolución de Wazuh

2009
2009

Nacimiento de OSSEC

Wazuh tiene sus raíces en OSSEC, un HIDS de código abierto creado por Daniel Cid. Se enfoca en detección de intrusiones y monitoreo de logs.

2015
2015

Fork y creación de Wazuh

Tras la adquisición de OSSEC por Trend Micro, se crea el fork Wazuh, añadiendo nuevas funcionalidades y mejorando la escalabilidad.

2017
2017

Integración con ELK Stack

Wazuh se integra completamente con Elastic Stack, proporcionando dashboards potentes y capacidades avanzadas de búsqueda y visualización.

2019
2019

Se añaden integraciones nativas con AWS, Azure y Google Cloud, permitiendo monitoreo unificado de infraestructuras híbridas.

# Ejemplo de regla de detección en Wazuh
<group name="linux,">
  <rule id="100100" level="10">
   <match>authentication failure</match>
   <description>Failed authentication attempt detected</description>
  </rule>
  <rule id="100101" level="12">
   <if_sid>100100</if_sid>
   <same_source_ip />
   <frequency>5</frequency>
   <timeframe>60</timeframe>
   <description>Multiple failed authentication attempts from same IP</description>
  </rule>
</group>
