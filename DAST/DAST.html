DAST

Dynamic Application Security Testing - Análisis dinámico de seguridad para aplicaciones en ejecución

Detecta vulnerabilidades en aplicaciones desplegadas en tiempo real

¿Qué es DAST?

DAST (Dynamic Application Security Testing) es una técnica de análisis de seguridad que prueba aplicaciones en ejecución desde el exterior, simulando ataques reales para identificar vulnerabilidades.

A diferencia de SAST, DAST analiza la aplicación desde "afuera hacia adentro", identificando problemas de seguridad que solo se manifiestan cuando la aplicación está en funcionamiento.

// DAST simularía ataques a endpoints en ejecución
GET /api/users?id=1' OR '1'='1
POST /login {
  "username": "admin' --",
  "password": "anything"
}
GET /profile/../etc/passwd

Ventajas Principales

  • Pruebas en entorno de producción o staging real
  • Detección de vulnerabilidades de runtime y configuración
  • No requiere acceso al código fuente
  • Identifica problemas de infraestructura y servidores
  • Detección de vulnerabilidades lógicas y de negocio
  • Compatible con cualquier tecnología o lenguaje

Técnicas de Ataque DAST

Métodos de escaneo utilizados:

  • Fuzzing de parámetros y entradas
  • Inyección SQL y NoSQL automatizada
  • Pruebas de Cross-Site Scripting (XSS)
  • Escaneo de directorios y archivos sensibles
  • Pruebas de autenticación y autorización
  • Análisis de headers de seguridad

Vulnerabilidades detectadas:

  • Inyecciones de código (SQL, OS, LDAP)
  • Cross-Site Scripting (XSS) reflejado y almacenado
  • Falsificación de peticiones (CSRF)
  • Exposición de datos sensibles
  • Configuraciones de seguridad incorrectas
  • Componentes con vulnerabilidades conocidas
// Ejemplo de ataque SQLi detectado por DAST
GET /products?category=Gadgets' UNION SELECT username, password FROM users--
// DAST identificaría esta inyección SQL y reportaría vulnerabilidad

DAST vs SAST: Comparación

DAST

  • Prueba aplicaciones en ejecución
  • No requiere código fuente
  • Detección de vulnerabilidades de runtime
  • Identifica problemas de configuración
  • Detección de vulnerabilidades lógicas
  • Compatible con cualquier tecnología

SAST

  • Analiza código fuente estático
  • Requiere acceso al código
  • Detección temprana en desarrollo
  • Identifica vulnerabilidades en código
  • Puede detectar problemas de diseño
  • Dependiente del lenguaje/framework
