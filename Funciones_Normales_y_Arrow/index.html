Funciones en JavaScript: Normales vs Arrow - Origen, Evolución y Aplicaciones Prácticas

Desde ES1 (1997) hasta ES2023: La evolución de las funciones
Diferencias entre funciones normales y arrow functions

Funciones Normales

Funciones tradicionales en JavaScript, definidas con la palabra clave function. Existen desde el inicio de JavaScript en 1995 y ofrecen características únicas como su propio contexto this.

Características Clave

  • Tienen su propio contexto this
  • Pueden usar arguments object
  • Pueden ser constructoras (usando new)
  • Tienen prototype property
Ejemplo:
// Función normal
function sumar(a, b) {
    return a + b;
}

// Constructor
function Persona(nombre) {
    this.nombre = nombre;
}

// Método
Persona.prototype.saludar = function() {
    console.log(`Hola, soy ${this.nombre}`);
};

Arrow Functions

Arrow functions fueron introducidas en ES6 (2015). Tienen una sintaxis más concisa y no tienen su propio this, arguments, super, o new.target.

Características Clave

  • No tienen propio this (heredan del contexto)
  • Sintaxis más corta y expresiva
  • No pueden ser constructoras
  • Return implícito en expresiones de una línea
Ejemplo:
// Arrow function básica
const sumar = (a, b) => a + b;

// Sin parámetros
const saludar = () => 'Hola Mundo';

// Con un solo parámetro (paréntesis opcional)
const cuadrado = x => x * x;

// En callbacks
const numeros = [1, 2, 3];
const cuadrados = numeros.map(n => n * n);

Historia de la Evolución

1995-1997

Nacimiento de JavaScript

Brendan Eich crea JavaScript. Las funciones normales son la única opción disponibles en ES1.

2009

ES5 y Strict Mode

Introducción del modo estricto que afecta el comportamiento de this en funciones.

2015

ES6: Arrow Functions

Revolución con arrow functions, clases, y muchas mejoras sintácticas.

2020-2023

ES2020+ y Mejoras

Optional chaining, nullish coalescing, y mejoras en funciones asíncronas.

Usar Funciones Normales cuando:

  • Necesitas usar this dinámico
  • Vas a crear constructores
  • Necesitas acceder a arguments
  • Definir métodos en objetos

Usar Arrow Functions cuando:

  • Trabajas con callbacks
  • Quieres mantener el contexto this
  • Necesitas funciones cortas y expresivas
  • Trabajas con arrays (map, filter, reduce)

Especificaciones Técnicas de Funciones

Funciones Normales

  • Palabra clave: function
  • Hoisting: Completo
  • Constructor: new Funcion()
  • Arguments: arguments disponible
  • Prototype: func.prototype

Arrow Functions

  • Sintaxis: () => {}
  • Hoisting: Solo si asignada a variable
  • Constructor: No permite new
  • Arguments: No disponible
  • Prototype: No tiene
# Ejemplo de Diferencia de this
function Persona() {
this.edad = 0;
// Arrow function mantiene contexto Persona
setInterval(() => {
this.edad++; // this = Persona
}, 1000);
}

Casos de Uso Prácticos

Mejor con Funciones Normales

  • Métodos en objetos y clases
  • Constructores y herencia prototípica
  • Funciones generadoras
  • Cuando necesitas arguments object
  • Event handlers con binding dinámico

Mejor con Arrow Functions

  • Callbacks y funciones anónimas
  • Array methods (map, filter, reduce)
  • Funciones que encapsulan this
  • Funciones de una línea con return implícito
  • Promesas y async/await
